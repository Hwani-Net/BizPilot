# /verify-and-report — 검증, 보고 및 오류 학습

코딩 작업 완료 후 반드시 실행하는 절차입니다.

## 1단계: 빌드 검증
// turbo
```
npm run build
```
- 빌드 실패 시 에러를 수정한 뒤 재시도

## 2단계: 기능 테스트
- 변경한 기능을 직접 실행하여 동작 확인
- 가능하면 브라우저/텔레그램으로 E2E 테스트

## 3단계: 결과 보고
사용자에게 아래 형식으로 보고:
```
✅ 완료 요약
- 변경 파일: [목록]
- 수행 내용: [설명]
- 테스트 결과: [성공/실패]
```

## 4단계: 오류 학습 기록 ⚠️ (중요)
**작업 중 에러가 발생했거나 예상과 다른 동작이 있었다면**, 반드시 전역 오류 기록 파일에 추가합니다:

파일 위치: `C:\Users\AIcreator\.agent\LESSONS_LEARNED.md`

기록 형식:
```markdown
## [날짜] [프로젝트명] 오류 제목
- **증상**: 어떤 에러가 발생했는가
- **근본 원인**: 왜 발생했는가
- **해결법**: 어떻게 해결했는가
- **예방법**: 다음에 같은 실수를 피하려면
- **관련 파일**: 영향받은 파일 목록

```

## 4.5단계: 성공 사례 기록 (5인 에이전트 체제 활용 시) 🏆
**난제 해결, 혁신적인 디자인, 시스템 아키텍처 개선** 등 5인 체제로 수행한 작업은 다음 파일에 기록하여 자산화합니다:

파일 위치: `C:\Users\AIcreator\.agent\SUCCESS_LOGS.md`

기록 형식:
```markdown
## [날짜] [프로젝트명] [카테고리] 성공 사례 제목
- **문제 정의(PM)**: 핵심 난제
- **전략 및 설계(Architect)**: 구조적 접근법
- **핵심 프롬프트/액션**: 사용했던 결정적인 프롬프트나 로직
- **결과 및 성과**: 정량적/정성적 성과 및 교훈
```
