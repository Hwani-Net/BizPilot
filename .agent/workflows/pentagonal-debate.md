# 🛡️ 5인 에이전트 처절한 토론 (Pentagonal Fierce Debate)

이 워크플로우는 NanoClaw의 5인 에이전트 시스템을 사용하여 안건에 대해 처절하게 토론하고 검증하는 절차입니다.
기존 Council 스킬보다 훨씬 공격적이고 비판적인 'Fierce Mode'가 적용됩니다.

## 1. 개요

| 역할 | 페르소나 | 핵심 질문 |
|---|---|---|
| **PM** | 냉철한 관리자 | "돈이 됩니까? 사용자가 원합니까?" |
| **Architect** | 타협 없는 설계자 | "기술 부채는? 확장은 가능한가?" |
| **Designer** | 완벽주의 디자이너 | "아름다운가? 'Wow' 포인트는?" |
| **Developer** | 현실적 개발자 | "구현 가능한가? 납기는?" |
| **Reviewer** | 비관적 QA | "버그는? 보안은? 최악의 경우는?" |

## 2. 사용 방법

### Telegram 또는 터미널

```bash
/debate "토론 주제"
```

**예시:**
- `/debate 블로그 자동 발행 시스템의 전면 개편`
- `/debate 리액트에서 스벨트로 마이그레이션`
- `/debate 신규 유료 멤버십 모델 도입`

### 실행 로직

1. **Round 1 (맹공)**: 5인 에이전트가 각자의 관점에서 무자비하게 비판합니다.
2. **Analysis**: 합의 점수(Consensus Score)를 계산합니다.
   - 점수가 너무 높으면(>70점), **Devil's Advocate**가 발동하여 외부 충격(예산 삭감, 경쟁사 출시 등)을 주입합니다.
3. **Round 2~3 (재반박)**: 외부 충격 상황에서 생존을 위한 대안을 제시하거나 서로를 공격합니다.
4. **Final Decision**: PM이 모든 비판을 수용하여 최종 실행 계획을 수립합니다.

## 3. 결과물

토론 로그는 `data/debate/debate_TIMESTAMP.json`에 저장되며, 채팅창에는 요약본이 출력됩니다.

## 4. 팁

- "처절한" 토론을 원한다면 주제를 구체적으로 잡으세요. (X: "AI 투자", O: "AI 투자 예산을 2배 늘리고 인력을 50% 감축하는 안")
- Reviewer의 지적은 뼈아프지만 반드시 해결해야 할 리스크입니다.
